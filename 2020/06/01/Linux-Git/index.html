Linux-Git

Linux

Linux文件基本属性

使用ll或者ls –l命令来显示一个文件的属性以及文件所属的用户和组

对于 root 用户来说，一般情况下，文件的权限对其不起作用
更改文件属性（略）

Linux系统目录结构：

/  系统根目录
./  当前目录
../  上级目录

Linux 文件与目录管理

绝对路径：
路径的写法，由根目录 / 写起，例如： /usr/share/doc 这个目录。
相对路径：
路径的写法，不是由 / 写起，例如由 /usr/share/doc 要到 /usr/share/man 底下时，可以写成： cd ../man
ls: 列出目录及文件名
cd：切换目录
pwd：显示目前的目录
mkdir：创建一个新的目录
rmdir：删除一个空的目录
cp: 复制文件或目录
rm: 移除文件或目录
mv: 移动文件与目录，或修改文件与目录的名称
man: 各个命令的使用文档

ls

选项与参数
-a ：全部的文件，连同隐藏文件( 开头为 . 的文件) 一起列出来(常用)
-d ：仅列出目录本身，而不是列出目录内的文件数据(常用)
-l ：长数据串列出，包含文件的属性与权限等等数据；(常用)

Linux指令

shutdown 关机指令，可以man shutdown 来看一下帮助文档。
shutdown –h now 立马关机
reboot 就是重启，等同于 shutdown –r now

[Tab]      ## 接在一串指令的第一个字的后面，则为『命令补全』
[Tab]      ## 接在一串指令的第二个字以后时，则为『文件补齐』

如果在Linux 底下输入了错误的指令或参数，想让当前的程序『停掉』的话，可以输入
[Ctrl] + c
键盘输入结束(End Of File, EOF 或 End Of Input),也可以用来取代 exit 的输入,例如想要直接离开文字接口可以输入
[Ctrl] + d ## 相当于输入 exit

cat  由第一行开始显示文件内容
tac  从最后一行开始显示，可以看出 tac 是 cat 的倒着写！

Git

Git一般工作流程

克隆 Git 资源作为工作目录。
在克隆的资源上添加或修改文件。
如果其他人修改了，你可以更新资源。
在提交前查看修改。
提交修改。
在修改完成后，如果发现错误，可以撤回提交并再次修改并提交。

Git常用命令

git init 初始化当前目录为一个Git仓库
git init newrepo 创建指定目录的Git仓库
git add filename 添加新文件
git commit -m "Adding files" 提交到仓库
